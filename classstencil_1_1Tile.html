Detailed Description

template<typename T, uindex_t width, uindex_t height, uindex_t halo_radius, uindex_t burst_length>
class stencil::Tile< T, width, height, halo_radius, burst_length >

A rectangular container of cells with a static size.

StencilStream tiles the grid and the ExecutionKernel works with those tiles: it receives the content of a tile (together with it's halo) and emits the contents of a tile. A tile is partitioned in four corner buffers, four border buffers and one core buffer. This is done to provide easy access to the halo of a tile. Have a look at the Architecture for more details of the data layout.

This tile manager supports copy operations to and from monolithic, user-supplied buffers, as well as an index operation to access the individual parts.

Copy the contents of a buffer into the tile.

This will take the buffer section [offset[0] : min(grid_width, offset[0] + tile_width)) x [offset[1] : min(grid_height, offset[1] + tile_height)) and copy it to the tile. This means that if the grid does not contain enough cells to fill the whole tile, those missing cells are left as-is.

Copy the contents of the tile to a buffer.

This will take the tile section [0 : min(tile_width, grid_width - offset[0])) x [0 : min(tile_height, grid_height - offset[1])) and copy it to the buffer section [offset[0] : min(grid_width, offset[0] + tile_width)) x [offset[1] : min(grid_height, offset[1] + tile_height)). This means that if the tile plus the offset is wider or higher than the grid, those superflous cells are ignored.

Parameters
bufferThe buffer to copy the data to.
offsetThe offset of the buffer section relative to the origin of the buffer.

cl::sycl::buffer<T, 2> stencil::Tile< T, width, height, halo_radius, burst_length >::operator[] ( Part  tile_part)
inline

Return the buffer with the contents of the given part.

If the part has not been accessed before, it will allocate the part's buffer. Note however that this method does not initialize the buffer. Please also note that the buffer is burst-aligned: The height of the returned buffer (the second value of the range) is always burst_length and the width is big enough to store all required cells of the part. For more information, read about Burst-aligned buffers.
