Detailed Description

template<typename T, uindex_t stencil_radius, typename TransFunc, uindex_t pipeline_length = 1, uindex_t tile_width = 1024, uindex_t tile_height = 1024, uindex_t burst_size = 1024>
class stencil::StencilExecutor< T, stencil_radius, TransFunc, pipeline_length, tile_width, tile_height, burst_size >

Execution coordinator.

The StencilExecutor binds the different parts of StencilStream together and provides a unified interface for applications. Users create a stencil executor, configure it and then use it to run the payload computations. It has multiple logical attributes that can be configured:

Grid

The grid is the logical array of cells, set with StencilExecutor::set_input A stencil executor does not work in place and a buffer used to initialize the grid can be used for other tasks afterwards. The StencilExecutor::run method alters the state of the grid and the grid can be copied back to a given buffer using StencilExecutor::copy_output.

SYCL queue

The queue provides the OpenCL platform, device, context and queue to execute the kernels. A stencil executor does not have a queue when constructed and StencilExecutor::run tries to select the FPGA emulation device if no queue has been configured yet. StencilExecutor::set_queue is used to directly set the queue, but StencilExecutor::select_emulator and StencilExecutor::select_fpga can be used to automatically configure the FPGA emulator or the FPGA, respectively.

Transition Function

A stencil executor stores an instance of the transition function since it may require some configuration and runtime-dynamic parameters too. An instance is required for the initialization, but it may be replaced at any time with StencilExecutor::set_trans_func.

Generation Index

This is the generation index of the current state of the grid. StencilExecutor::run updates and therefore, it can be ignored in most instances. However, it can be reset if a transition function needs it.

Template Parameters
TCell value type.
stencil_radiusThe static, maximal Chebyshev distance of cells in a stencil to the central cell. Must be at least 1.
TransFuncAn invocable type that maps a Stencil to the next generation of the stencil's central cell.
pipeline_lengthThe number of hardware execution stages. Must be at least 1. Defaults to 1.
tile_widthThe number of columns in a tile. Defaults to 1024.
tile_heightThe number of rows in a tile. Defaults to 1024.
burst_sizeThe number of bytes to load/store in one burst. Defaults to 1024.
